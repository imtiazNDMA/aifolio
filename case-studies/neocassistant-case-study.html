Case Study: neocAssistant

Context-aware disaster management assistant designed to support emergency operations with retrieval-backed responses and practical operator workflows.

FastAPI + React RAG + Local LLM Frontend + Backend Delivery

Problem

Emergency teams need rapid, context-aware answers from scattered guidance documents, but manual lookup slows incident response.

Architecture

Two-service architecture with a React web client and FastAPI backend. Backend handles retrieval workflow, session context, and model orchestration.

Constraints

  • Low-latency responses for field usage
  • Support for local inference stack
  • Simple operations via launcher scripts

Metrics

  • 2-service production-style split (frontend + backend)
  • Dedicated health/docs endpoints for operational visibility
  • RAG pipeline enabled for multi-document grounding

Your Role

Owned system design and implementation end to end: backend APIs, retrieval workflow, frontend UX, local model setup flow, and integration testing.
